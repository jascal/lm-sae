# `runs/` — result artifacts

Every script writes a small **`<name>_summary.json`** (the headline numbers) plus, in some
cases, larger artifacts (full per-head `.txt`/`.json` listings, `.npz` bundles, `.pt`
checkpoints).

**Layout.** Artifacts are organized into subdirectories that mirror the `scripts/` groups:

```
runs/
├── substrate/          ├── disassembly/        ├── gemma/
├── cov95_forge_tax/    ├── two_basis_forge/    ├── forge_smoke/
├── entanglement_tower/ └── tiny_gpt.pt   ← shared substrate checkpoint (git-ignored, stays at root)
```

**Git policy** (`.gitignore`): only `runs/**/*_summary.json` (at any depth) + this `README.md`
are tracked — they are small and are the data behind every table in
[`../README.md`](../README.md) and [`../docs/`](../docs/). Everything else under `runs/` (the
big disassembly `.txt`/`.json`, `*.log`, `tiny_gpt.pt`, the `forge_example/` tree, `data/*.npz`)
is **git-ignored** and regenerated by re-running the script.

The sections below map 1:1 to the subdirectories.
